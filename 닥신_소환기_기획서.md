# 🚗 닥신 소환기 — 기획서

> 엔카 중고차 링크를 넣으면, 닥신 AI agent를 소환하는 완성된 프롬프트를 뱉어주는 정적 웹사이트

---

## 📌 프로젝트 개요

**핵심 철학 (orrery 레퍼런스):**
- 서버 없음, 백엔드 없음 — 순수 브라우저 동작
- GitHub Pages / Vercel 무료 배포
- 사용자가 생성된 프롬프트를 복사해서 Claude/GPT에 직접 붙여넣기

---

## 🏗️ 아키텍처

```
사용자 입력
  └─ 엔카 링크 + 구매 조건 (예산 / 용도 / 연식 허용범위)
        ↓
[브라우저 내 로직]
  ├─ 링크에서 차량 ID 파싱
  ├─ 엔카 오픈 데이터 / Jina AI Reader로 페이지 스크래핑 (선택)
  └─ 프롬프트 조립기
        ↓
[출력]
  ├─ System Prompt (닥신 soul) — 복사 버튼
  └─ User Prompt (이 차 평가해줘 + 차량 데이터) — 복사 버튼
```

---

## 🧱 프롬프트 구조 (3-Layer)

### Layer 1: Soul Prompt (System) — 닥신의 정체성

> `steellife-ai-chat`의 soul 개념 차용

```
당신은 닥신입니다. 국내 중고차 시장 20년 경력의
전문 딜러 출신 AI입니다...

- 성격: 직설적, 전문적, 소비자 편
- 말투: 친근하지만 전문가다운
- 평가 기준: 시세 / 사고이력 / 주행거리 / 옵션 / 딜러 신뢰도
```

### Layer 2: Knowledge Prompt — 닥신의 판단 기준

> 유튜브 닥신 영상 기반

```
- 체크 항목 우선순위 (사고이력 > 침수 > 주행거리 > 가격)
- 엔카 딜러 등급 해석법
- 연식 / 가격 황금비율 기준표
- 절대 사면 안 되는 차 패턴
```

### Layer 3: Data Prompt (User) — 이 차의 실제 데이터

> 엔카 링크 파싱 결과

```
- 차종 / 연식 / 주행거리 / 가격 / 옵션
- 딜러 정보 / 지역
- 사용자 구매 조건 (예산, 용도, 협상 의사)

→ "위 정보를 바탕으로 이 차를 닥신 기준으로 평가해주세요"
```

## 📐 사이트 구조 (Pages)

```
/
├─ 메인 (링크 입력 + 조건 설정)
│   ├─ 엔카 URL 입력창
│   ├─ 구매 조건 폼 (예산, 용도, 협상 여부)
│   └─ "닥신 소환하기" 버튼
│
├─ 결과 화면
│   ├─ [복사] System Prompt — 닥신 soul
│   ├─ [복사] User Prompt — 차량 데이터 + 질문
│   ├─ [통합 복사] 한 번에 복사 (Claude Projects용)
│   └─ 사용 가이드 (어디에 붙여넣나)
│
└─ 닥신 소개 페이지
    └─ 닥신이 뭔지 + 유튜브 링크
```

## ⚙️ 기술 스택

| 항목 | 선택 | 이유 |
|---|---|---|
| 프레임워크 | **React + Vite** | orrery와 동일, 정적 배포 최적 |
| 스타일 | **Tailwind CSS** | 빠른 개발 |
| 배포 | **GitHub Pages** | 무료, 서버리스 |
| 엔카 데이터 | **Jina AI Reader** (`r.jina.ai/엔카URL`) | CORS 없이 클라이언트에서 페이지 읽기 가능 |
| 상태관리 | **useState만** | 단순 구조 |

## 🔑 핵심 기능 상세

### 엔카 링크 처리 방식 (2가지 선택지)

1. **자동 파싱** — `r.jina.ai` 경유로 엔카 페이지 텍스트 추출 → 프롬프트에 자동 삽입 *(더 편함)*
2. **수동 입력** — 링크만 넣고 나머지는 사용자가 직접 차량 정보 입력 *(더 단순, 확실)*

> → **1안 먼저 시도, 안 되면 2안 fallback** 구조 권장

### 프롬프트 출력 형태

- **Claude Projects용**: System + User 분리 출력
- **ChatGPT / 단일 채팅용**: 통합 하나짜리 출력
- 복사 후 붙여넣기 → 닥신이 바로 평가 시작

## 🗓️ 개발 단계

### Phase 1 (1~2일): 기본 뼈대
- Vite + React 프로젝트 세팅
- URL 입력 UI 구현
- 닥신 soul 프롬프트 초안 작성

### Phase 2 (2~3일): 핵심 로직
- 엔카 URL 파싱 (차량 ID 추출)
- Jina Reader로 차량 정보 자동 추출 시도
- 프롬프트 조립 로직 구현

### Phase 3 (1~2일): UI/UX 완성
- 복사 버튼 + 토스트 알림
- 사용 가이드 모달
- 모바일 반응형

### Phase 4 (1일): 배포
- GitHub Pages 배포
- 도메인 연결 (선택)

## 💡 차별화 포인트

- **닥신 soul** — 단순 "차 평가해줘"가 아니라 닥신의 말투, 판단기준, 우선순위가 녹아든 프롬프트
- **조건 맞춤** — 내 예산/용도를 넣으면 닥신이 "이 사람한테 이 차가 맞냐"는 관점으로 평가
- **즉시 사용** — AI에 붙여넣기만 하면 닥신이 바로 답변
