> **참고 프로젝트**

> - [tomeido/steellife-ai-chat](https://github.com/tomeido/steellife-ai-chat): 기업용 AI 챗봇 (Next.js, A2A 프로토콜)

## 1. 프로젝트 개요 및 핵심 가치

**목표**  
사용자가 '엔카' 중고차 매물 링크를 입력하면, AI 에이전트(닥신)가 해당 차량을 평가하는 데 최적화된 **프롬프트를 생성**해 주는 웹사이트를 구축한다.

**핵심 가치**  
- 복잡한 프롬프트 작성 없이 누구나 'AI 닥신'이라는 일관된 페르소나의 전문적인 차량 평가를 경험  
- orrery처럼 **프롬프트 생성에 집중**하고, 사용자는 생성된 프롬프트를 자신이 원하는 AI(ChatGPT, Gemini 등)에 붙여넣기만 하면 됨  
- '닥신' 특유의 친근하고 전문적인 말투와 평가 기준을 프롬프트에 내장

**주요 사용자**  
중고차 구매를 고려 중이지만 차량 지식이 부족하여 객관적인 평가를 원하는 일반인

## 2. 개발 단계별 상세 계획

### 2.1 1단계: 기획 및 데이터 분석 (MVP 범위 설정)

- **URL 파싱 로직 설계**  
  - 엔카 상세 페이지 URL 패턴 분석 (예: `encar.com/.../detail.do?carid=123456`) → 차량 고유 ID 추출
- **정보 추출 항목 정의 (MVP)**  
  - 공개된 페이지에서 보이는 기본 정보: 제조사, 모델, 연식, 주행거리, 사고 이력 유무, 가격 등  
  - (고도화) 추후 실제 크롤링 또는 API 연동을 통해 더 정확한 데이터 확보

### 2.2 2단계: 프롬프트 엔지니어링 ('AI 닥신' 페르소나 및 평가 기준)

- **페르소나 정의**  
  - 유튜브 '닥신' 캐릭터 분석: 전문적이면서도 친근한 말투, 비유를 곁들인 설명, '내돈내산 원칙', '허위매물 거르기' 등 핵심 가치 주입
- **평가 항목 도출 (프롬프트 내 구조화)**  
  1. **가격 대비 상태** – 시세 비교  
  2. **사고/침수 이력 리스크**  
  3. **옵션 및 성능**  
  4. **연식/주행거리 대비 향후 유지비**  
  5. **총평 및 추천 여부**  
- **프롬프트 구조화**  
  - AI가 1단계에서 추출한 정보를 바탕으로 위 항목을 일관되게 평가하도록 명확한 지시사항과 출력 포맷(JSON, 마크다운 등)을 포함한 템플릿 작성

### 2.3 3단계: 프론트엔드 개발 (UI/UX 및 프롬프트 생성 기능)

- **기술 스택**  
  - Next.js + TypeScript + Tailwind CSS (orrery와 steellife-ai-chat 참고, SEO 및 배포 용이)
- **핵심 UI 컴포넌트**  
  - 입력 필드: "엔카 링크를 붙여넣으세요"  
  - 생성 버튼: "AI 닥신 프롬프트 생성하기"  
  - 결과 표시 영역: 생성된 프롬프트를 보여주는 텍스트 영역 + **'복사' 버튼** (orrery의 "AI 에이전트용 텍스트 복사" 기능 차용)
- **프롬프트 생성 로직 (브라우저 내 동작)**  
  1. 사용자 입력 URL → 차량 ID 파싱  
  2. (MVP) 파싱된 ID와 함께 2단계의 마스터 프롬프트 템플릿을 결합하여 최종 프롬프트 동적 생성  
  3. (고도화) 추후 서버 사이드에서 실제 엔카 페이지 크롤링 또는 API 연동하여 더 정확한 데이터를 프롬프트에 포함

### 2.4 4단계: 백엔드 및 인프라 (선택적, 고도화 단계)

- **데이터 수집 에이전트**  
  - steellife-ai-chat의 A2A 개념 응용, 백엔드에서 엔카 페이지 HTML 분석 → 상세 정보(사고 이력 상세, 옵션 코드 등) 추출
- **프롬프트 최적화 API**  
  - 단순 템플릿 결합을 넘어 수집된 데이터 기반 정교한 프롬프트 생성 API 제공
- **대시보드/로그**  
  - steellife-ai-chat의 `/logs` 페이지처럼 요청 통계(인기 링크, 사용량 등)를 볼 수 있는 관리자 페이지 고려
