# AI 닥신 프롬프트 사이트 기획서

## 1. 프로젝트 목표

- 사용자가 ChatGPT, Claude, 기타 LLM에 **직접 복붙해서 쓸 수 있는 “닥신 중고차 평가 프롬프트”**를 자동으로 생성해주는 웹 서비스 구축.
- 서비스 자체는 **LLM API를 호출하지 않고**, `엔카 링크 → 프롬프트 텍스트`만 제공하는 구조를 지향(혼천의/orrery처럼 브라우저에서만 동작하는 계산기·도구 컨셉).[^1][^2]
- 닥신TV의 중고차 고르는 핵심 로직(보험 이력, 성능기록부, 용도 변경, 1인 소유, 엔카 진단, 가격 대비 감가 등)을 프롬프트에 녹여서, LLM이 엔카 페이지를 닥신 스타일로 분석하게 만드는 것이 핵심.[^3][^4][^5][^6]

## 2. 레퍼런스 정리

### 2.1 rath/orrery에서 가져올 포인트

- 혼천의(사주팔자·자미두수·출생차트 계산기)처럼 **입력 폼 + 결과 영역 + “AI 해석용 전체 텍스트” 탭 구조**를 사용.[^1]
- 서버 없이 브라우저에서 동작하는 계산기 형태(정적 페이지 + JS)라는 점을 그대로 가져와서, 이번 서비스도 **GitHub Pages/Vercel 등 정적 호스팅으로 동작** 가능하게 설계.[^2][^1]

### 2.2 닥신TV 콘텐츠에서 가져올 로직

닥신TV의 여러 영상과 2차 정리 글들을 보면, 좋은 중고차를 고를 때 반복적으로 강조하는 기준이 있다.[^4][^5][^6][^3]

- **보험 이력(카히스토리)**
  - 용도 변경 이력: 렌트/영업(대여) 이력이 있으면 초보자는 피하라고 강조.[^5][^6][^4]
  - 소유자 변경 이력: 1인 소유(1인 신조)를 가장 좋게 봄.[^6][^3][^4][^5]
  - 전손/침수 이력: 무조건 거르는 조건.[^4][^5][^6]
  - 사고 금액: 100–300만 원대 국산 기준은 가벼운 판금·도색 수준으로 보는 편, 500만 원 이상 대형 수리는 리스크로 취급.[^3][^5][^6]
  - 정보 비공개/비공개 구간: 보험 이력·성능 기록부 중 하나라도 비공개면 더 볼 것도 없이 건너뛰라고 설명.[^5][^6]

- **성능 점검 기록부**
  - 외판(펜더, 문짝, 범퍼 등) 교환·판금은 크게 문제 삼지 않지만, **인사이드 패널·휠하우스·프레임(뼈대) 교환은 거르는 조건**으로 둠.[^6][^3][^4][^5]

- **엔카 진단**
  - 엔카 진단까지 받은 차량은 보험 이력 + 성능 기록부 + 엔카 진단의 삼중 검증을 통과한 셈이라 신뢰도 플러스 요인으로 판단.[^4][^5]

- **가격/감가 로직**
  - “새 아반떼 가격이면 감가 많이 된 그랜저·수입 대형차도 볼 만하다”는 식으로, **동일 예산에서 한 급 위 차종을 노리는 전략**을 긍정적으로 평가.[^3][^5][^6]

이 핵심 기준들을 프롬프트에 그대로 녹여서, LLM이 엔카 상세 페이지를 읽으면서 같은 체크리스트로 판단하도록 만드는 것이 목표다.

## 3. 서비스 기능 정의

### 3.1 핵심 시나리오

1. 사용자가 웹 페이지를 연다.
2. 엔카 차량 상세 페이지 URL을 붙여 넣는다.
3. (옵션) 예산, 사용 용도(출퇴근/가족/장거리), 연식·연비 선호도, 닥신 말투 강도(순한/그대로)를 선택한다.
4. “프롬프트 생성” 버튼을 누르면, 아래에 **LLM에게 그대로 붙여 넣을 수 있는 긴 프롬프트 텍스트**가 생성된다.
5. 사용자는 이 텍스트를 ChatGPT/Claude/기타 BYOK 프론트엔드(TeaTree Chat, TypingMind 등)에 복붙하고, 그 안에서 결과를 확인한다.[^7][^8]

### 3.2 페이지 구성

- 상단 헤더
  - 서비스 이름(예: “닥신 프롬프트 공장” – 가칭)
  - 간단한 설명 문구
  - GitHub 링크(오픈소스 저장소)

- 메인 입력 영역
  - 엔카 URL 입력 인풋
  - 옵션 토글들
    - 타겟 LLM: ChatGPT / Claude / 기타
    - 출력 스타일: 요약 위주 / 디테일 위주 / 투자 관점(감가·가성비)
    - 말투: 닥신 스타일에 가깝게 / 부드러운 전문가 톤

- 결과 영역
  - 탭 1: “전체 프롬프트” – LLM에 붙여 넣을 긴 텍스트
  - 탭 2: “간단 프롬프트” – 시스템 프롬프트를 짧게 하고, 나머지는 LLM 자율에 맡기는 버전
  - 탭 3(옵션): “프롬프트 구조 설명” – 개발자용 문서/해설

- 푸터
  - 면책 조항(이 서비스는 법적/정비적 책임을 지지 않으며, 최종 결정은 사용자가 직접 해야 한다는 안내)

## 4. 아키텍처 및 기술 스택

### 4.1 기본 방향

- **완전 정적 SPA**
  - Next.js(정적 export) 또는 Astro/Vanilla Vite + React 등 선택
  - 백엔드는 필수 아님(프롬프트 생성은 순수 클라이언트 로직 + 템플릿 문자열 조합)

- **호스팅**
  - GitHub Pages, Vercel, Netlify 등 중 하나 선택
  - rath/orrery처럼 GitHub Pages 기반 정적 배포도 가능.[^2][^1]

- **상태 관리**
  - 단일 페이지 내에서 React useState 정도면 충분
  - 프롬프트 템플릿을 JSON/TS 객체로 정의해두고, 옵션에 따라 조합

### 4.2 확장 여지

- 나중에 원하면 별도 서버(Cloudflare Workers, Vercel Edge Functions) 추가해서
  - 간단한 클릭 통계/프롬프트 버전 A/B 테스트
  - 자주 쓰는 프롬프트를 짧은 코드로 축약해서 제공(예: `/p/short?id=...`)

## 5. AI 닥신 Soul 프롬프트 설계

### 5.1 설계 원칙

- 실제 인물 “신재욱(닥신TV)”을 그대로 사칭하기보다는, **“닥신TV의 중고차 철학을 학습한 가상의 전문가”**로 정의해서 법적·윤리적 리스크를 줄이는 방향.
- 닥신이 강조하는 규칙(1인 소유, 용도 변경 없음, 외판·뼈대 구분, 비공개 이력 거르기, 감가·가성비 관점 등)을 시스템 프롬프트 안에 규칙 리스트로 명시.[^5][^6][^3][^4]
- LLM이 엔카 링크를 직접 열 수 있는지 여부에 따라 두 가지 버전 제공:
  - **브라우징 가능 모델용**: “이 URL을 열고 페이지 내용을 분석하라”
  - **브라우징 불가 모델용**: “URL 페이지의 텍스트/스크린샷을 사용자가 추가로 붙여 넣게 한다”

### 5.2 프롬프트 구조(초안)

1. 시스템 역할 정의(닥신 스타일 전문가)
2. 입력으로 받을 것 명시
   - 엔카 링크(또는 페이지 텍스트)
   - 사용자의 예산/용도/리스크 선호도
3. 분석 체크리스트
   - 보험 이력, 성능 기록부, 엔카 진단, 가격/감가, 타이어·딜러 성향 등[^3][^4][^5]
4. 출력 포맷
   - 요약(한글 5줄 이내)
   - 리스크 체크리스트(불릿)
   - 닥신 스타일 코멘트(말투 옵션 반영)
   - 최종 결론: “사도 됨 / 애매 / 비추” + 이유

### 5.3 사이트에서 뱉어줄 예시 프롬프트(브라우징 가능 LLM용)

아래는 사이트가 최종적으로 생성해줄 프롬프트 예시(엔카 URL과 옵션은 런타임에 치환):

```text
[시스템]
너는 한국 중고차 시장에 매우 익숙한 중고차 전문 컨설턴트다. 
유튜브 ‘닥신TV’에서 설명하는 중고차 고르는 원칙을 참고하되, 실제 인물을 사칭하지 말고 그와 비슷한 기준으로 합리적으로 판단하는 전문가로 행동한다.

분석 기준은 다음과 같다.
1. 보험 이력(카히스토리)
   - 용도 변경 이력에 대여/영업 사용 이력이 있는지 확인하고, 초보자에게는 이런 차를 피하라고 조언한다.
   - 소유자 변경 이력이 1회(1인 소유)인지, 여러 번 바뀌었는지 본다.
   - 전손, 침수, 도난 이력이 있으면 무조건 비추천한다.
   - 내차 피해·타차 가해 사고 금액을 보고, 국산차 기준 100–300만 원대의 소액 사고는 판금·도색 수준으로 보고, 500만 원 이상 대형 사고는 리스크로 본다.
   - 보험 이력이나 성능 기록부가 비공개이거나, 정보 제공 불가능 기간이 길면 바로 리스크가 크다고 판단한다.

2. 성능 점검 기록부
   - 외판(펜더, 문짝, 범퍼 등) 교환/판금은 큰 문제가 아니라고 보되,
   - 인사이드 패널, 휠하우스, 프레임(뼈대) 교환/수리가 있으면 큰 사고차로 보고 비추천한다.

3. 엔카 진단
   - 엔카 진단을 받았는지, 외판·뼈대에 이상 없음으로 나오는지 확인하고, 삼중 검증(보험 이력 + 성능 기록부 + 엔카 진단)을 통과하면 신뢰도를 높게 본다.

4. 가격/감가 및 차종 포지셔닝
   - 같은 예산에서 새 아반떼 vs 감가 많이 된 그랜저·수입 대형차처럼, 한 급 위 차종을 살 수 있는지 본다.
   - 연식, 주행거리, 감가율을 함께 보고 가성비를 평가한다.

5. 기타 신호
   - 타이어 브랜드(미쉐린 등 프리미엄 vs 싸구려)로 차주의 관리 성향을 추정한다.
   - 딜러의 다른 매물, 누적 판매 이력, 엔카 진단 비율 등을 통해 딜러 성향을 추정하고, 지나치게 리스크 높은 매물만 모아 파는 딜러는 경계한다.

위 기준을 일관되게 적용해서, 감정적인 표현보다 논리적인 근거를 중심으로 설명해라.

[사용자 입력]
- 엔카 차량 링크: {{ENCA_URL}}
- 내 정보/조건:
  - 예산: {{BUDGET}}
  - 주요 용도: {{USAGE}} (예: 출퇴근, 가족용, 장거리 위주 등)
  - 리스크 허용도: {{RISK_LEVEL}} (보수적 / 보통 / 공격적)

[지시]
1. {{ENCA_URL}} 페이지를 직접 열어, 보험 이력, 성능 기록부, 엔카 진단, 가격, 연식, 주행거리, 타이어, 딜러 정보를 가능한 한 많이 읽어라.
2. 위에서 정의한 닥신 스타일 기준으로 이 매물을 평가해라.
3. 아래 형식으로 한국어로만 답변해라.

[출력 형식]
1) 한줄 요약
- 이 차를 한 줄로 요약하라. (예: “감가 많이 된 1인 소유 그랜저, 리스크 낮은 편”) 

2) 핵심 스펙 요약
- 차종 / 연식 / 주행거리 / 연료 / 트림
- 신차 대비 추정 감가율

3) 리스크 체크리스트
- 보험 이력: 용도 변경 / 소유자 수 / 사고 금액 / 비공개 여부
- 성능 기록부: 외판 교환 여부, 뼈대 손상 여부
- 엔카 진단: 유무 및 결과
- 기타: 타이어, 딜러 성향 등

4) 닥신 스타일 코멘트
- 닥신TV에서 설명하는 톤을 참고하되, 실제 인물임을 암시하지 말고 가상의 전문가로서 직설적으로 평가하라.
- 왜 괜찮은지, 왜 애매한지, 왜 위험한지 구체적으로 설명하라.

5) 최종 결론
- 아래 중 하나를 선택하고, 한 문장으로 명확히 써라.
  - “이 조건이면 사도 괜찮은 차다.”
  - “아주 싸게만 산다면 고려할 수 있다.”
  - “초보자에게는 비추다. 다른 매물을 보는 것이 좋다.”
- 그 판단의 핵심 근거를 2–3줄로 정리하라.
```

사이트에서는 위 텍스트에서 `{{ENCA_URL}}`, `{{BUDGET}}` 등 플레이스홀더를 실제 입력값으로 치환한 후 결과 영역에 표시하면 된다.

## 6. 개발 단계별 로드맵

### 6.1 1단계 – 최소 기능 버전(MVP)

- [ ] 프로젝트 스캐폴딩
  - Vite + React or Next.js 생성, 기본 레이아웃 구축
- [ ] UI 초안
  - 엔카 URL 입력 필드 + “프롬프트 생성” 버튼
  - 결과 텍스트 영역 + “복사하기” 버튼
- [ ] 프롬프트 템플릿 하드코딩
  - 위 예시 프롬프트를 JS 문자열로 넣고, URL만 치환
- [ ] GitHub Pages 또는 Vercel에 배포

### 6.2 2단계 – 옵션화 및 템플릿 엔진화

- [ ] 옵션 추가
  - 말투(직설/순한), 출력 길이, 타겟 모델 등 선택 UI
- [ ] 템플릿 엔진 레이어
  - 단순 `string.replace`에서, 옵션에 따라 블록을 켜고 끄는 형태로 리팩토링
  - 예: `sections.daksinStyleCommentStrong` / `sections.daksinStyleCommentSoft`
- [ ] “간단 프롬프트” 탭 추가
  - 시스템 프롬프트를 짧게 줄이고, LLM이 스스로 기준을 정리하도록 유도하는 버전

### 6.3 3단계 – 개발자 경험 및 문서화

- [ ] 프롬프트 구조 설명 탭
  - 각 섹션이 닥신 영상의 어느 부분 로직을 반영했는지 한글로 주석 설명.[^4][^5][^3]
- [ ] GitHub README 작성
  - rath/orrery처럼 사용 방법, 호스팅 방법, 기여 방법 정리.[^1][^2]
- [ ] 버전 관리
  - `v0.1` : URL만 치환

### 6.4 4단계 – 향후 확장 아이디어

- **여러 매물 비교용 프롬프트**
  - 엔카 링크를 2–3개 받아서, “같은 예산에서 어떤 게 닥신 기준으로 더 낫냐” 비교하는 프롬프트 생성
- **차종별 프리셋**
  - 국산 준중형 / 국산 대형 / 수입 디젤 대형 / EV 등 테마별로 닥신이 선호하는 감가 패턴을 약간씩 달리 반영
- **사용자 정의 규칙 추가**
  - 예: “무조건 1인 소유만”, “주행 10만 이상은 제외” 같은 개인 룰을 프롬프트에 자동 삽입

## 7. 정리

- 서비스는 **엔카 링크를 넣으면 닥신 스타일의 중고차 평가 프롬프트를 만들어 주는 정적 웹 도구**다.
- rath/orrery의 구조(입력 폼 + 결과 + AI 해석용 탭)와 닥신TV의 중고차 필터링 로직을 결합하여, 사용자가 원하는 LLM 어디에나 붙여 넣어 쓸 수 있는 형태로 설계한다.[^6][^1][^5][^3][^4]
- 초기에는 브라우징 가능한 LLM을 전제로 설계하고, 추후 필요시 브라우징 불가 모델용 템플릿도 추가한다.
